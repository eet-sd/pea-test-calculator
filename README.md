# ค่าทดสอบเพื่อการตรวจรับพัสดุ กฟภ. (PEA Acceptance Test Calculator)

เว็บแอปแบบ static สำหรับตอบคำถาม: พัสดุรหัสนี้/ชื่อนี้ ซื้อจำนวนเท่านี้ → ต้องทดสอบหัวข้ออะไรบ้าง สุ่มกี่ตัวอย่าง ค่าทดสอบประมาณเท่าไร

- **ไม่มี backend, ไม่เก็บข้อมูลผู้ใช้** — เปิดไฟล์ `index.html` ตรงๆ ก็ใช้งานได้
- แหล่งข้อมูลเดียว: ชีตจำนวนสุ่ม `13-O-DYPJ_T98LJRIByvGRT82gF1lWfpizEcOzvwaOR4` (213 รหัส) ซึ่งเป็นฐานของ dashboard [PEA Acceptance Test Laboratory](https://datastudio.google.com/u/1/reporting/cb1b305f-ef53-4d0e-9d32-b2e08f932f63/page/LpktB) — ทั้งหัวข้อทดสอบ ราคา จำนวนสุ่ม ที่มา และลิงก์สเปค
- เงื่อนไขพิเศษรายหัวข้อ (เช่น hotline clamp สุ่ม 10 → Torque 6 / Temp rise 4, OES/Hardness/Dropping point ทดสอบ 1 ตัวอย่าง) อยู่ใน `rules.js`

## โครงสร้างไฟล์

| ไฟล์ | หน้าที่ |
|---|---|
| `index.html` | หน้าเว็บ + ตรรกะคำนวณทั้งหมด |
| `data.js` | ตารางพัสดุ 213 รหัส พร้อมกติกาสุ่มที่ parse แล้ว (generate จากสคริปต์ — อย่าแก้มือ) |
| `rules.js` | เงื่อนไขพิเศษรายหัวข้อทดสอบ **แก้ไข/เพิ่มด้วยมือได้** (มีคำอธิบายในไฟล์) |
| `tools/convert_data.py` | สร้าง `data.js` ใหม่จาก export ของชีตจำนวนสุ่ม |

## อัปเดตข้อมูล

**การแก้ชีตไม่ได้อัปเดตหน้าเว็บอัตโนมัติ** — หน้าเว็บใช้ snapshot ที่ฝังใน `data.js` ต้องรันสคริปต์อัปเดต:

- **ดับเบิลคลิก `update_data.bat`** — ดาวน์โหลดชีตล่าสุด → สร้าง `data.js` → commit + push ขึ้น GitHub Pages ให้ครบในคลิกเดียว
- การดาวน์โหลดอัตโนมัติต้องแชร์ชีตเป็น *Anyone with the link (Viewer)*; ถ้าไม่อยากเปิด ให้ดาวน์โหลด CSV เอง (File → Download → CSV) บันทึกเป็น `tools\sheet_latest.csv` แล้วรัน `update_data.bat` ซ้ำ
- ตัวแปลง (`tools/convert_data.py`) parse คอลัมน์จำนวนสุ่มทุกแบบ (ช่วงจำนวน, ค่าคงที่, √3 ของล็อต, 5%, E1/E2, "ไม่กำหนด") และพิมพ์คำเตือนถ้าแถวไหน parse ไม่ได้ — แถวนั้นจะกลายเป็นโหมดกรอกจำนวนสุ่มเอง ไม่พังทั้งเว็บ

## ข้อจำกัด

- ราคา/จำนวนสุ่มเป็นไปตาม snapshot ของชีต — ตรวจสอบกับหน่วยงานทดสอบก่อนใช้อ้างอิงจริง
- รหัสที่ชีตระบุจำนวนสุ่ม "ไม่กำหนด" หน้าเว็บจะให้กรอกจำนวนสุ่มเอง
- กรณีสุ่มเกิน 3 ตัวอย่างของกลุ่ม VT/CT มีเงื่อนไขเพิ่มเติมในสเปค (แสดงไว้ในข้อความกติกาแล้ว แต่ไม่คำนวณให้)
